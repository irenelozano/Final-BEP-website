## 1. What This Project Does

This project uses the **DSMC (Direct Simulation Monte Carlo)** method via the **SPARTA** open-source code to compute the **aerodynamic drag force** on spherical tin micro-particles traveling through a low-pressure hydrogen buffer gas. This is directly relevant to **EUV lithography** at ASML, where tin debris ejected during plasma generation must be decelerated before it can damage sensitive collector optics.

**The physics in one sentence:** A tin sphere (100 nm–10 μm) sits in a 100 Pa, 293 K hydrogen stream flowing at 20–700 m/s. Every case is **free-molecular** (Knudsen number Kn ≈ 12–1230), meaning gas molecules travel many droplet diameters between collisions, so continuum fluid dynamics (Navier-Stokes) is invalid and a kinetic (molecule-by-molecule) treatment is required.

**The pipeline in one sentence:** A bash script computes optimal numerical parameters (grid size, timestep, particle count) for each (diameter, velocity) pair, calls SPARTA to run the DSMC simulation, parses the force output, and logs results to a CSV. MATLAB scripts then validate against analytical theory (Epstein, Baines, Stokes, Cunningham) and generate trajectory/penetration analyses.

## 3. Prerequisites & Installation

### 3.1 System Requirements

- Linux (tested on Ubuntu 22.04)
- C++ compiler with C++11 support (gcc ≥ 9)
- MPI (OpenMPI ≥ 4.0 or MPICH)
- CMake ≥ 3.10
- MATLAB ≥ R2020b (for post-processing)
- ffmpeg (optional, for video)
- bash, awk

### 3.2 Installing SPARTA (from source)

```bash
# Step 1: Clone SPARTA from the official Sandia repository
git clone https://github.com/sparta/sparta.git
cd sparta

# Step 2: Build with CMake (recommended)
mkdir build && cd build
cmake ../cmake -DCMAKE_BUILD_TYPE=RELEASE
make -j$(nproc)

# The binary is at: build/sparta
# You can verify it works:
./sparta -h
```

**Alternative: Classic make build** (if CMake is unavailable):

```bash
cd sparta/src
make mpi -j$(nproc)
# Binary at: src/spa_mpi
```

### 3.3 Configuring the SPARTA Binary Path

All pipeline scripts hard-code a SPARTA path like:

```bash
/home/ilozano/sparta/src/spa_mpi
```

**You MUST update this path** in every script you intend to run. There are two ways:

**Option A — Symlink (recommended, one-time setup):**

```bash
mkdir -p ~/sparta/src
ln -s /path/to/your/sparta/build/sparta ~/sparta/src/spa_mpi
```

**Option B — Edit each script:** Change the line in every `run_*.sh` script:

```bash
# Find this line:
mpirun -np 4 /home/ilozano/sparta/src/spa_mpi ...

# Replace with:
mpirun -np 4 /path/to/your/sparta/build/sparta ...
```

## 4. Global Assets: The Physical Model

This section explains what every number in the global asset files means. You should understand this before running any simulation.

### 4.1 `hydrogen.species`

```
H2 2.016 3.346e-27 2 0.2 2 1.9e-5 6331.0 1.0 0
```

| Field | Value | Meaning |
|-------|-------|---------|
| Name | H2 | Species identifier |
| Mol. weight | 2.016 | Atomic mass units (amu) |
| Mass (kg) | 3.346e-27 | Molecular mass of H₂ |
| Rot. DOF | 2 | Diatomic molecule: 2 rotational degrees of freedom |
| Rot. relax | 0.2 | Rotational collision number (20% of collisions relax rotation) |
| Vib. DOF | 2 | Vibrational DOF (inactive at 293 K — too cold) |
| Vib. relax | 1.9e-5 | Vibrational collision number (effectively infinite at 293 K) |
| Char. vib. T | 6331.0 | Characteristic vibrational temperature of H₂ (K) |
| Species type | 1.0 | Standard gas species |
| Flags | 0 | No special treatment |

The key physics: at 293 K, H₂ has fully excited translation and rotation but **no vibrational excitation** (vibrational modes need T ≫ 6331 K).

### 4.2 `hydrogen.vhs`

```
H2 2.72e-10 0.67 273.15 1.0
```

| Field | Value | Meaning |
|-------|-------|---------|
| Species | H2 | Must match species file |
| d_ref (m) | 2.72e-10 | VHS reference diameter at T_ref |
| ω | 0.67 | VHS viscosity exponent (ω=0.5 = hard sphere, ω=1.0 = Maxwell) |
| T_ref (K) | 273.15 | Reference temperature |
| ε/k_B (K) | 1.0 | VHS well depth (not used for VHS, placeholder) |

This VHS model determines the mean free path:

λ = 1 / [√2 · n · π · d_ref² · (T/T_ref)^{ω-0.5}]

At 100 Pa, 293 K: λ ≈ 123 μm.

### 4.3 `data.sphere` (Surface Mesh)

A unit sphere represented by 98 points and 192 triangles (4 subdivisions per face = 4×4 quads on 6 cube faces mapped to sphere). The projected-area error is 0.55%, verified in the report.

At runtime, SPARTA scales it by `diam_scale` and translates it to `(Lx/2, Ly/2, Lz/2)` via:

```sparta
read_surf ${mesh_file} scale ${diam_scale} ${diam_scale} ${diam_scale} ftrans 0.5 0.5 0.5
```

Two larger meshes are available (not used in production):
- `data_1200.sphere` — 1200 triangles
- `data_2400.sphere3d` — 2400 triangles

## 5. The Master SPARTA Input Template (`in.drag`)

This is the core of the simulation. Located at `pipeline/in.drag`.

### 5.1 How Parameter Injection Works

All case-specific parameters are passed via SPARTA's `-var` command-line flag:

```bash
mpirun -np 4 spa_mpi \
    -var diam_scale 100e-9 \
    -var vel_val 700 \
    -var fnum_val 3 \
    -var grid_res 40 \
    -var t_step 5.84431e-11 \
    -var TRANSIENT_STEPS 25000 \
    -var STEADY_STEPS 25000 \
    -var out_name "output.txt" \
    < in.drag
```

Inside `in.drag`, `${diam_scale}` is replaced with `100e-9`, `${vel_val}` with `700`, etc.

### 5.2 Line-by-Line Explanation of `in.drag`

```sparta
# 1. SIMULATION INITIALIZATION
seed 12345             # Random number seed (fixed for reproducibility)
dimension 3            # 3D simulation
global gridcut 0.0 comm/sort yes  # No cell cutting, enable MPI communication sorting

# 2. DOMAIN & GRID
boundary oo rr rr      # x: outflow/outflow, y: reflect/reflect, z: reflect/reflect
                       # "o" = molecules leave freely, "r" = molecules reflect specularly
create_box 0 ${Lx} 0 ${Ly} 0 ${Lz}   # Box from (0,0,0) to (50d, 50d, 50d)
create_grid ${grid_res} ${grid_res} ${grid_res}   # Cartesian grid, N³ cells
balance_grid rcb cell  # Recursive coordinate bisection load balancing

# 3. GAS PROPERTIES
global nrho 2.47e22 fnum ${fnum_val}  # Number density n = P/(k_B·T) = 100/(1.38e-23·293)
                                       # fnum = macroparticle weight (each sim particle represents fnum real molecules)
species ../global_assets/hydrogen.species H2   # Load H2 molecular model
collide vss all ../global_assets/hydrogen.vhs  # VHS collision model for all species

# 4. FLOW INJECTION
mixture inlet H2 temp 293.0 vstream ${vel_val} 0.0 0.0  # Inlet gas: H2 at 293K, velocity in +x
fix in emit/face inlet xlo  # Inject molecules from the x=0 plane

# 5. SPHERE SURFACE
read_surf ${mesh_file} scale ${diam_scale} ${diam_scale} ${diam_scale} ftrans 0.5 0.5 0.5
                           # Read sphere mesh, scale to droplet size, centre in box
surf_collide 1 cll 293.0 0.3 0.3 0.3 0.0  # CLL model: Twall=293K, α_n=α_t=α_rot=0.3, α_vib=0.0
surf_modify all collide 1  # All surfaces use collision model 1

# 6. COMPUTES & FIXES
compute ID_drag_compute surf all all fx fy fz  # Per-triangle force on all surfaces

# PHASE 1: TRANSIENT (flow establishment, no measurement)
run ${TRANSIENT_STEPS}

# PHASE 2: STEADY-STATE SAMPLING
fix f_drag_cumul ave/surf all 1 ${STEADY_STEPS} ${STEADY_STEPS} c_ID_drag_compute[*]
    # Cumulative average = sample every step (N=1), average over STEADY_STEPS,
    # output once at end (Nfreq = STEADY_STEPS)

fix f_drag_blocks ave/surf all 1 1000 1000 c_ID_drag_compute[*]
    # Block averages every 1000 steps (for convergence diagnostics)

dump 1 surf all ${STEADY_STEPS} ${out_name} id f_f_drag_cumul[*]
    # Dump cumulative average once at end of steady window
dump 2 surf all 1000 ${out_name}_blocks id f_f_drag_blocks[*]
    # Dump block averages every 1000 steps

run ${STEADY_STEPS}     # Run the steady-state sampling phase
```

### 5.3 The CLL Surface Model

```
surf_collide 1 cll 293.0 0.3 0.3 0.3 0.0
```

This sets:
- **Twall = 293 K** — droplet surface temperature (isothermal, cold assumption)
- **α_n = 0.3** — normal accommodation coefficient (30% diffuse, 70% specular)
- **α_t = 0.3** — tangential accommodation coefficient
- **α_rot = 0.3** — rotational accommodation coefficient
- **α_vib = 0.0** — vibrational accommodation (inactive at 293 K)

Values of α:
- α = 0 → fully specular reflection (no momentum transfer to surface)
- α = 1 → fully diffuse reflection (maximum momentum transfer, thermalised)

The chosen α = 0.3 is an **assumption** — no measured H₂–Sn accommodation data exist. This is the single largest uncertainty in absolute drag magnitude.

## 7. Step-by-Step: Post-Processing in MATLAB

### 7.1 Before You Start

1. **Ensure a production matrix CSV exists** in `post_processing/`. Either:

   - Copy an existing CSV from `production_output/` to `post_processing/`

### 7.2 Script-by-Script Guide

#### 7.2.1 Plot 30-Case Drag Matrix (`plot_matrix_results.m`)

**What it produces:** Figure 5.1 in the report — two-panel log-log plot of drag force:
- Panel (a): Drag vs diameter at each fixed velocity
- Panel (b): Drag vs velocity at each fixed diameter

**How to run:**
```matlab
>> cd /path/to/sparta/simulations/tin_drag/post_processing
>> plot_matrix_results
```

**What it does internally:**
1. Reads the CSV
2. Fits a linear model F(v) per diameter to extract F₀ (intercept at v=0)
3. Subtracts F₀ from all raw drag values
4. Removes any zero/negative forces
5. For each velocity, fits power law F ∝ d^n
6. For each diameter, fits power law F ∝ v^m
7. Creates a publication-quality figure

**If the script errors:**
```
Error using readtable
File 'matrix_results_20260601_125031.csv' not found.
```
→ Update the `filename` variable on line 8 to match your CSV name.
#### 7.2.2 Theoretical Validation (`theoretical_validation.m`)

**What it produces:** Figure 5.2 — 4-panel figure comparing DSMC drag against Epstein, Baines, Stokes, and Cunningham models at v = 20, 100, 550, 700 m/s.

**How to run:**
```matlab
>> theoretical_validation
```

**What it does internally:**
1. Loads DSMC data with F₀ correction (same as above)
2. Computes Epstein drag: F_Ep = (π/2) · d² · ρ_gas · v · c̄ · (1 + α·π/8) where c̄ = √(8k_BT/πm) is mean thermal speed
3. Computes Baines drag: generalised free-molecular resistance valid at finite speed ratio S
4. Computes Stokes drag: F_St = 3πμdv (no-slip continuum, expected to fail badly)
5. Computes Cunningham drag: Stokes × slip correction factor C(Кn)
6. Plots all on log-log axes

**Key physics the figure demonstrates:**
- Stokes overpredicts by 1–4 orders of magnitude → continuum invalid
- Cunningham is within factor ~2 → slip correction helps but still imperfect
- DSMC follows F ∝ d² (same slope as Epstein/Baines) → correct kinetic scaling
- ~49% drag deficit at large d, high v (visible as DSMC below Epstein line)

#### 7.2.3 Force vs Velocity Validation (`fdrag_Velocity_validation.m`)

**What it produces:** Figure 5.3 — two-panel figure of F vs v:

#### 7.2.5 Stopping Distance Analysis (`stopping_distance_analysis.m`)

**What it produces:** Figure 6.1 — Velocity decay vs distance for ≤ 1 μm droplets. Shows reference distances at 480 mm and 1300 mm.

```matlab

5. Plots velocity vs distance curves
6. Reports stopping distances at reference planes

## 10. Understanding the Output Files

### 10.1 SPARTA Force Dump Format

Files like `force_D1e-06_V700_20260615_123456.txt` use SPARTA's dump format:

```
ITEM: TIMESTEP
25000
ITEM: NUMBER OF SURFACES
192
ITEM: SURFACES id f_f_drag_cumul[1] f_f_drag_cumul[2] f_f_drag_cumul[3]
1 -3.4567e-20  1.2345e-21  5.6789e-21

```

- Column 1: triangle ID
- Column 2: time-averaged Fx on that triangle (N)
- Column 3: time-averaged Fy (N)
- Column 4: time-averaged Fz (N)

The total drag force is the sum of column 2 across all 192 triangles.

### 10.2 SPARTA Log Format

Files like `log_D1e-06_V700_20260615_123456.log` contain:

```
...
Step CPU Np Ncoll Nscoll
25000 12.34 1234567 987654 321
26000 12.56 1234567 987654 321
...
```

- Np: number of simulated particles
- Ncoll: number of intermolecular collisions this step
- Nscoll: number of surface collisions this step

After the runs, timing info:
```
Total CPU time = 1234.56 seconds
Total wall time = 345.67 seconds
```

### 10.3 Steady-State Proof Timeseries

Format of `ss_proof_d10um_v700_20260615_123456_timeseries.txt`:

```
ITEM: TIMESTEP
200
ITEM: NUMBER OF SURFACES
192
ITEM: SURFACES id f_f_drag_blocks[1] f_f_drag_blocks[2] f_f_drag_blocks[3]

Each ITEM:TIMESTEP block = one data point (average over RECORD_EVERY steps).

## 12. Known Limitations

From Chapter 7 of the report. Critical for anyone using these results:

### 12.1 Accommodation Coefficient (α = 0.3)

**Problem:** The CLL coefficients α_n = α_t = α_rot = 0.3 are assumed, not measured. No experimental H₂–Sn accommodation data exist for these conditions.

**Impact:** This is the single largest uncertainty in absolute drag magnitude. If actual α is higher, drag is higher → stopping distances are shorter. If α is lower, drag is lower → stopping distances are longer.

**Recommendation:** An α-sensitivity study (α ∈ [0.1, 1.0]) would convert single-point estimates into defensible ranges.

### 12.2 Isothermal, Rigid, Inert Droplet

**Problem:** The droplet is modelled as a rigid sphere at 293 K. Real tin micro-particles originate molten (Tmelt ≈ 505 K), which changes the momentum of reflected molecules. Evaporation, deformation, and SnH₄ surface chemistry are neglected.

**Impact:** Stopping distances could be different for hot (real) vs cold (simulated) particles. The hot-wall effect would increase drag (hotter surface → faster reflected molecules → more momentum transfer).

### 12.3 Quasi-Steady Drag

**Problem:** Each DSMC run computes steady drag at fixed v. The trajectory model applies this steady law to a decelerating particle. Unsteady / added-mass effects during deceleration are excluded.

**Justification:** Timescale separation: τ_gas ~ 10⁻⁷ s ≪ τ_droplet ~ 10⁻³–10⁻² s.

### 12.4 1D Single-Particle Trajectory

**Problem:** Only axial drag is modelled. No angular distribution, no particle-particle interactions, no bulk sweeping flow. Results are a "worst-case axial estimate".

**Impact:** Real debris capture fraction in the chamber will be higher (more effective) due to the sweeping flow.

### 12.5 Under-Resolved Large Droplets

**Problem:** The 10 μm production case uses N=10 (Δx = 50 μm), which:
- Exceeds the Bird criterion: Δx < λ/3 ≈ 41 μm
- Is coarser than the grid convergence plateau (N ≈ 34)

**Mitigation:** Grid coarsening was shown to **raise** the computed drag (wrong direction to explain the ~49% deficit), so the deficit is physical, not numerical. But the 5–10 μm results are "bounds, not fully grid-converged production points".

### 12.6 No Experimental Validation

**Problem:** All validation is against analytical references (Epstein, Baines). No experimental deceleration measurements exist.

**Impact:** The ~49% large-droplet offset remains unresolved between:
- Genuine finite-Knudsen physics (the most likely explanation)
- Differences between CLL kernel and the analytical diffuse/specular blend

## 13. `in.drag` vs `in.steadystate_proof` vs `validation_in.drag`

| Feature | `in.drag` | `in.steadystate_proof` | `validation_in.drag` |
|---------|-----------|------------------------|-----------------------|
| **Purpose** | Production runs | Steady-state proof | α=1 control case |
| **Phase structure** | Phase 1 (transient) + Phase 2 (steady) as separate `run` commands | Single continuous `run` | Phase 1 + Phase 2 |
| **Force recording** | Starts after Phase 1 (only steady window) | Starts at step 0 (transient + steady) | Starts after Phase 1 |
| **Output** | Cumulative force + block averages | Time series every RECORD_EVERY steps + cumulative | Same as `in.drag` plus global force |
| **Accommodation** | α = 0.3 (from `surf_collide` in template) | α = 0.3 | α = 1.0 (hard-coded) |
